# Proof Library Skill

## Objective
Provide a dedicated source for Yuvabe's verified client work, case-study material, and reusable proof content without mixing it into writing or visual-direction skills.

## Use This Skill When
- Pulling proof points, credibility signals, outcomes, or service evidence for the website.
- Looking for examples of past work, capabilities, or offer framing from named Yuvabe projects.
- Building case studies, services pages, trust sections, or proof-heavy landing pages.
- Supporting copywriting with factual source material rather than writing rules.

## Workflow
1. Load `../brand-foundation/SKILL.md` first if the task affects public-facing messaging.
2. Start with `references/client-projects.md` for distilled, reusable proof from named client/project pages.
3. Open `references/project-source-links.md` when you need the original project URLs or need to check source confidence.
4. Distill only the relevant proof for the task at hand; do not dump large amounts of source content into the response.
5. Pair with `../copywriting/SKILL.md` for messaging tasks or `../marketing-website/SKILL.md` for section/page planning.

## Guardrails
- Treat this skill as a knowledge source, not a writing-style guide.
- Prefer verifiable, concrete claims over inflated summaries.
- Use named-client and project-page evidence before generic studio claims.
- If a project entry is marked partial or pending, do not invent detailed claims; use only what is explicitly captured.
- If source content conflicts with the current studio positioning, align the presentation with `brand-foundation` while preserving factual accuracy.
